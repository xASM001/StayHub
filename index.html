‚ùì Frequently Asked Questions

Q: Is StayHub really free?

A: Yes! StayHub is released under the GNU General Public License v3.0. You can use, modify, and distribute it freely without any licensing fees.

Q: What are the minimum system requirements?

A: Minimum 4GB RAM, 100GB storage, Windows/Linux/macOS. A database server (MySQL or PostgreSQL) is required. See System Requirements for details.

Q: Does it support multiple properties?

A: Phase 1 supports single property management. Multi-property support for hotel chains is planned for Phase 2.

Q: Can I use this for my commercial hotel?

A: Absolutely! GPL v3.0 permits commercial use. You can run it as-is or customize it for your business needs.

Q: How often are updates released?

A: We release monthly updates with bug fixes and minor features. Major releases happen quarterly with significant new functionality.

Q: Do you offer technical support?

A: Yes! Check our documentation, visit the community forum, search GitHub issues, or contact us at abdullah.s.mohammed51@gmail.com.

Q: Can I contribute to the project?

A: Absolutely! We welcome contributions from developers, designers, and documentation writers. See the Contributors page for guidelines.

Q: Is my data secure?

A: StayHub uses industry-standard security practices. Since you control the server, your data never leaves your infrastructure. See our Security documentation for details.

Architecture Documentation

System Architecture

StayHub follows a three-tier architecture pattern:

Presentation Layer: Web-based UI built with HTML5, CSS3, and JavaScript. Provides intuitive interfaces for managers, receptionists, and staff.

Business Logic Layer: Spring Boot backend with RESTful API endpoints. Handles all business rules, validations, and data processing.

Data Layer: Relational database (MySQL/PostgreSQL) with proper indexing and relationships. Ensures data integrity and performance.

Key Components

Controllers: Handle HTTP requests and route them appropriately
Services: Contain business logic and coordinate between layers
Repositories: Manage database operations using JPA
Entities: Define data models and relationships

Security

Role-based access control (RBAC)
Password encryption and hashing
SQL injection prevention through parameterized queries
HTTPS support for secure communications

Development Setup

Prerequisites

Java Development Kit (JDK) 11 or higher
Maven 3.6+ for dependency management
MySQL Server 5.7 or PostgreSQL 10+
Git for version control
IDE: IntelliJ IDEA or VS Code

Setup Steps

  1. Clone the repository: git clone https://github.com/xASM001/StayHub.git
  2. Navigate to project: cd StayHub/backend
  3. Install dependencies: mvn install
  4. Create database: CREATE DATABASE stayhub_db;
  5. Configure database in application.properties
  6. Run tests: mvn test
  7. Start application: mvn spring-boot:run

Testing

Run unit tests with: mvn test

Run integration tests with: mvn verify

Generate coverage report: mvn jacoco:report
